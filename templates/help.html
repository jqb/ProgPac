Basic Steps How to start and not hurt your self

There is only tree atomic moves:

  • s (step forward): slides the bug one field forward
  • r (turn right): turns the bug right
  • l (turn left): turns the bug left
  • So knowing this is not to hard to discover that code ssrsls will result as shown in image.

Functions Call the same thing more than once

Creating code only from atomic moves could quite hard to understand and not very useful if you want to beat code counter. Most case maps contained in games are some sort of symmetric structures which means that your code in most case could be re-used. Instead using Ctrl-C and Ctrl-V operation we will functions. 

    f:sssr
    lf

Analyzing this code from second line. First element is well known atomic step l (turn left) which means that first action taken by bug will be turning left. Next step is f which is not a valid step, but in first line holds a definition of this not atomic step (function). So what will is f call will replaced by it's content in f definition.

Function have to be named with use of single lowercase letter which is not s,r or l.

You are able to call defined function as many times and define as many functions as you need (well not quite because Latin alphabet has 24 characters and 3 of them are used for atomic moves, but it is still a lot). 

    f:sssr x:lsss rxff

This example will result as 

    rlssssssrsssr
